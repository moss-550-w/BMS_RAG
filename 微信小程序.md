# BMS领域专业RAG问答系统 微信小程序设计方案
结合微信小程序**轻量便捷、碎片化使用、移动端交互**的特点，在保留项目核心功能（零幻觉学术问答、精确文献溯源、双引擎支持）的基础上，针对移动端场景进行轻量化适配与交互优化，重点满足**快速查询、随时溯源、简洁配置**的需求。

---

## 一、整体设计定位
1. **目标用户**：BMS领域科研人员、工程师，需在实验室、通勤等场景下快速查询专业问题
2. **风格基调**：延续PC端工业科技风，适配移动端做轻量化调整——低饱和科技蓝为主色，浅灰背景，卡片式布局，突出内容核心
3. **核心设计原则**：
   - 极简操作：减少层级，核心功能（问答）一键触达
   - 内容优先：小屏幕下突出回答与引用，弱化装饰性元素
   - 场景适配：支持语音输入、问答分享、收藏等移动端特色功能
4. **色彩规范**（与PC端一致，保证品牌统一）
   - 主色：`#165DFF`（科技蓝，按钮、引用角标、选中态）
   - 辅助色：`#0FC6C2`（溯源高亮、收藏标记）
   - 中性色：`#F5F7FA`（页面背景）、`#333333`（正文）、`#86909C`（辅助文字）

## 二、小程序页面架构（底部Tab导航+核心页面）
采用**底部Tab导航**作为主框架，3个核心Tab页覆盖全部功能，层级不超过3层，保证移动端操作流畅。

```
┌─────────────────────────────────────────┐
│           顶部导航栏（全局）              │
│  页面标题 | 系统状态 | 更多操作（⋮）    │
├─────────────────────────────────────────┤
│                                         │
│          核心内容区（动态切换）          │
│  ┌──────────┐  ┌──────────┐  ┌───────┐ │
│  │  问答页  │  │ 文献库   │  │ 我的   │ │
│  │ (Tab1)   │  │ (Tab2)   │  │ (Tab3) │ │
│  └──────────┘  └──────────┘  └───────┘ │
│                                         │
├─────────────────────────────────────────┤
│           底部Tab导航栏                  │
│  [问答]  [文献库]  [我的]               │
└─────────────────────────────────────────┘
```

## 三、核心页面详细设计
### Tab1：问答页（核心首页，默认打开）
**功能目标**：快速发起问答、查看对话历史、实时展示溯源引用
#### 页面布局（从上到下）
1. **顶部状态区**
   - 系统状态：`向量库已加载`/`模型运行中`（轻量提示，不占空间）
   - 快捷操作：点击右上角「⋮」，弹出「清空对话」「帮助说明」
2. **对话展示区**（核心，占页面70%高度）
   - 气泡式对话：
     - 用户提问：浅灰色右对齐气泡，支持文字/语音输入标识
     - AI回答：白色左对齐卡片，**流式逐字输出**（适配移动端等待体验）
   - 回答结构化展示：
     - 核心结论：加粗高亮，第一时间呈现
     - 详细阐述：折叠/展开设计（默认展开，长内容自动折叠）
     - 引用角标：`[论文1, P5]` 形式，点击弹出溯源详情
   - 专业术语：自动标蓝高亮，点击可查看术语解释（扩展功能）
3. **底部输入区**（固定在页面底部）
   - 语音输入按钮：左侧麦克风图标，支持语音转文字（适配移动端场景）
   - 多行输入框：中间区域，支持换行、长文本输入
   - 发送按钮：右侧主色高亮「发送」，点击发起问答
   - 快捷重写：输入框上方「一键重写为学术式」按钮，点击优化问题

#### 特色交互：引用溯源弹窗
点击回答中的`[论文1, P5]`角标，从底部弹出半屏模态框：
- 顶部：论文标题、作者、页码
- 中间：原文片段预览（高亮匹配内容）
- 底部：「复制引用」「查看完整论文」（跳转至文献库详情页）

---

### Tab2：文献库页
**功能目标**：查看已加载论文、搜索论文、查看论文引用情况
#### 页面布局
1. **顶部搜索栏**
   - 搜索框：输入论文标题/关键词，实时过滤列表
   - 筛选按钮：按「引用次数」「加载时间」排序
2. **论文列表区**
   - 卡片式列表，每篇论文一个卡片：
     - 左侧：论文缩略图标（电池/BMS相关）
     - 中间：论文标题（截断显示，点击展开）、作者、引用次数
     - 右侧：箭头图标，点击进入详情页
3. **底部统计区**
   - 轻量提示：`已加载8篇BMS论文`/`FAISS索引已持久化`

#### 论文详情页（二级页面）
- 论文基础信息：标题、作者、发表年份、全文页码范围
- 引用片段列表：展示该论文被问答引用过的所有片段，点击可跳转至对应问答
- 操作按钮：「收藏论文」「分享论文信息」

---

### Tab3：我的页
**功能目标**：配置系统参数、查看对话历史、管理收藏
#### 页面布局（分组卡片式）
1. **配置管理组**
   - 向量引擎切换：单选框（火山引擎Ark/阿里云DashScope），点击即时生效
   - 检索参数：Top-K数值滑块（默认5，范围1-10）、Rerank开关（默认开启）
   - 提示：`文本分片固定250（已适配API限制）`
2. **个人数据组**
   - 对话历史：点击进入历史列表，支持按时间/关键词搜索，点击可恢复对话
   - 我的收藏：收藏的问答/论文列表，支持取消收藏
3. **系统信息组**
   - 系统状态：向量库状态、模型状态、版本号（V1.0）
   - 帮助说明：常见问题、使用指南
   - 关于项目：项目介绍、报告生成日期（2026-04-26）

## 四、核心业务流程设计
### 1. 标准问答流程（移动端优化版）
```
用户打开小程序 → 进入问答页（Tab1）
    ↓
输入方式选择：
  - 文字输入：直接在输入框打字
  - 语音输入：点击麦克风→语音转文字→确认内容
    ↓
可选：点击「一键重写」优化问题
    ↓
点击「发送」→ 前端校验问题非空
    ↓
调用后端API：POST /api/v1/query（携带问题、Top-K、引擎参数）
    ↓
后端处理：查询重写→FAISS检索→Rerank→大模型生成
    ↓
前端接收：流式逐字渲染回答
    ↓
用户查看回答：
  - 点击引用角标→弹出溯源弹窗
  - 点击术语→查看解释（扩展）
  - 点击「收藏」→保存至我的收藏
```

### 2. 对话历史恢复流程
```
进入「我的」页 → 点击「对话历史」
    ↓
搜索/筛选历史对话 → 点击某条对话
    ↓
跳转至问答页 → 恢复该对话的上下文
    ↓
可继续提问或查看引用
```

## 五、技术架构设计
### 1. 小程序前端技术选型
- **开发框架**：微信原生小程序开发（WXML/WXSS/JS）
  - 优势：性能最优，完全适配微信生态，无需跨端兼容
- **网络请求**：`wx.request`（HTTPS），对接后端FastAPI
- **流式输出**：WebSocket（`wx.connectSocket`），实现逐字渲染
- **数据存储**：`wx.setStorageSync`（本地缓存），存储对话历史、用户配置

### 2. 后端适配调整
- 保持原有FastAPI架构不变，新增小程序专属接口（或复用现有接口，兼容移动端参数）
- 优化API响应速度：压缩返回数据量，优先返回核心结论，详细内容按需加载
- 鉴权简化：采用API Key鉴权（存储在后端，小程序通过请求头传递），无需微信登录（适配内部使用场景）

### 3. 数据流架构
```
小程序前端
    ↓ ↑
HTTPS/WebSocket
    ↓ ↑
后端FastAPI（复用原有RAG核心引擎）
    ↓ ↑
FAISS向量库 + 本地文件系统
```

## 六、非功能性设计
### 1. 性能优化
- **对话懒加载**：历史对话仅加载最近20条，上滑加载更多
- **图片压缩**：论文缩略图、图标使用WebP格式，压缩至100KB以内
- **缓存策略**：缓存常用问题的答案，30分钟内重复提问直接返回本地结果
- **流式输出优化**：WebSocket连接复用，减少握手开销

### 2. 可靠性设计
- **API重试机制**：网络失败时自动重试2次（指数退避），失败后友好提示「网络异常，请稍后再试」
- **数据备份**：对话历史本地缓存+云端同步（可选，需后端支持）
- **错误降级**：Rerank不可用时，自动关闭并提示「重排服务暂不可用，已使用基础检索」

### 3. 安全性设计
- **HTTPS通信**：所有网络请求强制使用HTTPS
- **敏感信息隔离**：API Key存储在后端，小程序不存储任何敏感配置
- **输入校验**：前端+后端双重校验用户输入，防止注入攻击
- **数据隐私**：本地缓存数据加密存储，小程序退出时可选择清除

### 4. 微信生态适配
- **分享功能**：支持分享问答/论文至微信好友/群，分享卡片显示核心内容
- **收藏功能**：微信收藏+小程序内部收藏双重支持
- **消息通知**：可选：索引重建完成后发送微信服务通知（需后端对接微信模板消息）

## 七、小程序核心优势
1. **轻量便捷**：无需安装，随开随用，适配移动端碎片化查询场景
2. **核心功能完整**：保留PC端零幻觉问答、精确溯源、双引擎切换等核心能力
3. **交互优化**：语音输入、折叠/展开、溯源弹窗等设计，适配小屏幕操作
4. **生态融合**：支持微信分享、收藏，方便团队协作交流
5. **无缝对接**：后端复用原有RAG核心引擎，开发成本低，上线快
